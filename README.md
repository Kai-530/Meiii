## 功能特色

### 戰棋海克斯選擇系統
- **高度擬真客戶端**：從 **10 個專屬客製化海克斯** 中隨機抽取 **3 個** 進行初始展示，完美還原《英雄聯盟：聯盟戰棋》選擇介面。
- **獨立刷新機制**：每張卡片備有獨立的「刷新」按鈕，限制 **1 次獨立刷新**。配有防重複連點與指針阻斷保護（`pointer-events: none`），防止池空崩壞。
- **鎖定與視覺反饋**：選定海克斯後卡片亮起、其餘變暗鎖定，選擇結果會同步渲染於最後星球頁的左側面板中。

### 智慧音效控制系統
- **全站音效反饋**：刷新卡片、鎖定海克斯、開啟神秘禮物、翻開今日運勢、切換頁面均備有專屬動態音效。
- **音樂消長機制 (Ducking Effect)**：在吹蠟燭等關鍵音效播放時，系統會自動智慧調降背景生日歌音量，播放完畢後自動恢復，營造極佳的聽覺層次感。
- **全域喇叭開關**：首頁設有喇叭控制鈕，可隨時手動開啟或靜音，並在音效播放完畢後自動重設狀態。

### 蛋糕吹蠟燭儀式
- **動態蠟燭火焰**：採用細緻的 CSS 火焰抖動動畫，並配有提示氣泡對話框。
- **許願互動**：點擊蛋糕觸發 **3 秒許願倒數**，火焰熄滅，隨即於蠟燭頂部噴發 **物理粒子煙火特效**，帶來極具儀式感的吹蠟燭體驗。

### 互動式好友星軌祝福系統
- **動態橢圓星軌**：中央恆星（壽星）動態計算正弦與餘弦公式，讓 **4 位摯友星球** 沿著各自精準的橢圓軌道實時繞行。
- **好友在線名單**：右側面板展示好友在線列表，配有動態在線綠燈。好友與星座羈絆皆支援 **Hover Tooltip 浮動資訊框**。
- **星座羈絆系統**：將好友星座包裝成戰棋的**特性羈絆 (Traits)**（如小獅子 1/1、小雙子 1/2），趣味性滿分。
- **語音祝福卡片**：點擊好友星球彈出精緻卡片，並**實時播放該位好友親自錄製的生日語音**。

### 運勢抽籤系統
- **3D 空間翻牌**：採用 CSS 3D 空間翻轉特效（`backface-visibility` 與 `transform-style: preserve-3d`），模擬真實翻牌手感。
- **吉不吉運勢**：隨機抽取大吉、中吉、小吉等運勢，點擊後翻開，每次重返首頁時自動重置。

### 神秘禮物盒
- **隨機驚喜開箱**：點擊禮物盒觸發驚喜音效，彈出卡片展示 10 種隨機禮物描述與表情包。
- **狀態封存保存**：開啟後禮物盒會變暗封存，關閉彈窗後禮物盒轉換為表情包展示。

### 生日倒數與歡慶霓虹彩盒
- **精準計時鎖定**：鎖屏畫面精準倒數距離 **8 月 9 日** 的剩餘時間，支援 URL 參數 `?test=true` 進行解鎖測試。
- **歡慶霓虹特效**：生日當天，計時框會變身為粉、紫、金三色交織的**霓虹呼吸框**，充滿慶典氛圍。

### 行動裝置警告防禦
- **佈局防護**：檢測寬度小於 1000px 的螢幕時，自動切換為行動版防禦警告，提示「請使用電腦開啟」，保護寬螢幕遊戲客戶端佈局不被擠壓。

## 亮點與技術優化

### 1. 統一彈窗重置與狀態清除系統（Unified Modal Reset System） 
> [!IMPORTANT]
> **為解決點擊外圍遮罩關閉彈窗時產生的狀態殘留 Bug 進行的架構優化。**
- **問題背景**：原先點擊遮罩外圍會直接修改 DOM 的 `display='none'`，導致大星球彈窗寫入的 `width: 960px` 等 inline styles 殘留，影響後續小星球彈窗的佈局；且大星球音樂及彈幕定時器也會在關閉後繼續在後台播放。
- **最佳實踐**：在 `galaxy.js` 中實作了 `closeAnyModal()` 統一重置函數。無論是點擊「確認」還是「遮罩外圍」，均會統一進行：
  1. 暫停並重置所有朋友語音與背景音樂。
  2. 清除並重設彈幕定時器。
  3. 徹底清空 `.modal-card` 與 `#mBody` 的 inline style 屬性，防範樣式殘留污染其他彈窗版面。

### 2. 鎖屏樣式模組化抽離（Lockscreen Style Modularization）
> [!TIP]
> **提升代碼維護性與首頁防閃爍攔截的雙贏重構。**
- **重構方案**：將原本放置在 `index.html` 內部的 70 餘行鎖屏呼吸、時鐘指針與倒數震動等 CSS 樣式，獨立抽離至 `css/components/lockscreen.css` 中，並透過 `main.css` 統一管理。
- **效能考量**：保留了約 15 行的**同步鎖定檢查 JS 腳本**於 `index.html` 頂部。該腳本會在瀏覽器載入 `<body>` 前第一時間將 `is-locked` 標記寫入 `<html>`，完美避免了使用外部非同步腳本可能導致的「首頁先閃爍出現在畫面上，才跳出鎖屏」的視覺瑕疵（FOUC 閃爍防禦）。

### 3. 動態等比例自適應縮放（Viewport Auto-Scaling）
- **技術痛點**：本專案包含精準的橢圓軌道座標、大量絕對定位的星球以及 CSS 3D 空間卡片，若使用傳統 CSS 響應式佈局，極易在不同視窗大小下發生排版崩潰或重疊。
- **解決方案**：採用 `resize.js` 的 **JS 動態等比縮放技術**。自動以 1920px 寬度為設計基準，根據用戶當前螢幕尺寸動態為 `document.body` 賦予 `zoom` 縮放值（限制在安全區間 `0.5` ~ `1.5`），完美確保在任何螢幕解析度或系統縮放比（如 125%、150%）下，畫面皆保持一致的極致比例。

### 4. 分頁切換暫停與垃圾回收機制（Visibility Shield）
- **解決痛點**：防止用戶將網頁切換至後台分頁時，定時器排隊導致的 DOM 堆積與彈幕播放崩潰 Bug。
- **最佳實踐**：實時監聽 `visibilitychange` 事件。當用戶切換到其他分頁時，會**立刻清除彈幕定時器並清空舞台 DOM 元素**，防止後台堆積阻塞；用戶切換回來時則重新啟動，確保網頁永遠流暢。

### 5. 圖層疊加與 Z-Index 梳理
- **重構方案**：全面整理全站的 `z-index` 圖層樹。將通用彈窗遮罩 `.modal` 的 `z-index` 全域調升至最高層級（`9999`），根治了原先在大螢幕下，右上角計時框或音樂控制鈕會穿透覆蓋到彈窗最前方的視覺 bug，實現無懈可擊的圖層覆蓋體驗。
